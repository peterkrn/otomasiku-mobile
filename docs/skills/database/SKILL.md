## Monetary Values — CRITICAL

```prisma
// ✅ ALWAYS: BigInt for money
price          BigInt    // 19800000 = Rp 19.800.000
total_amount   BigInt
shipping_cost  BigInt

// ❌ NEVER: Float, Decimal, or Int for money
price          Float     // WRONG
price          Decimal   // WRONG
```

- Store as **integer Rupiah** (no decimals) — `Rp 19.800.000` = `19800000`
- Prisma returns `BigInt` → Express converts to string in JSON → Flutter parses to `int`

## Transactions — CRITICAL

Order creation MUST be atomic:

```typescript
// ✅ ALWAYS: Single prisma.$transaction for order flow
const order = await prisma.$transaction(async (tx) => {
  // 1. Validate stock
  // 2. Create order + order_items
  // 3. Deduct stock for each item
  // 4. Clear user's cart

});

// ❌ NEVER: Separate queries outside a transaction
const order = await prisma.order.create(...);    // WRONG if stock deduction is separate
await prisma.product.update(...);                // WRONG — not atomic
```

## Product Schema (Current)

```prisma
model Product {
  id             String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  name           String
  slug           String   @unique
  sku            String?  @unique
  description    String?
  brand          String                        // "Mitsubishi" | "Danfoss"
  series         String?                       // "FR-A800", "FC300", "MR-J4"
  sub_series     String?                       // "FR-A840", "FC 302"
  variant        String?                       // "2.2kW / 400V"
  price          BigInt
  original_price BigInt?
  stock          Int      @default(0)
  category       String                        // "Inverter" | "PLC" | "HMI" | "Servo" | "Module"
  unit           String   @default("pcs")
  min_order      Int      @default(1)
  images         String[]                      // primary image first, then gallery
  is_published   Boolean  @default(true)
  created_at     DateTime @default(now())
  updated_at     DateTime @updatedAt
  // ...relations
  @@map("products")
}
```

## Seeding

### Seed Script Pattern

```typescript
// prisma/seed.ts
import { PrismaClient } from '@prisma/client';
const prisma = new PrismaClient();

async function main() {
  // Seed 125 products from DUMMY_PRODUCT_MAPPING_PLAN.md
  await prisma.product.createMany({
    data: products,
    skipDuplicates: true,
  });
}

main()
  .catch(console.error)
  .finally(() => prisma.$disconnect());
```

### Product Seed Source

All 125 products are defined in `docs/DUMMY_PRODUCT_MAPPING_PLAN.md` § 3A–3E with:
- Product ID, SKU, name, brand, series, variant
- Price (in Rupiah integer), stock count
- Image file paths

## Database Functions (SQL)

Two auto-triggers exist (defined in migration):

1. **`generate_order_number()`** — auto-creates `OMA-YYYYMMDD-NNN` on order insert
2. **`update_updated_at()`** — auto-updates `updated_at` timestamp on row change

---

## Rules

1. **NEVER write raw SQL.** Use Prisma for everything.
2. **NEVER skip migrations.** Every schema change needs `prisma migrate dev`.
3. **NEVER seed test data in production.** Use environment checks.
4. **ALWAYS use `@db.Uuid` for ID fields** — matches Supabase `auth.users.id` format.
5. **ALWAYS verify ownership** in queries: `WHERE user_id = authenticatedUserId`.
